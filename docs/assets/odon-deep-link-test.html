Odon Deep Link Test

Use this page to test whether your operating system has registered odon:// links and whether Odon can open a saved project ROI from a clicked report link. This page does not upload or inspect local files; it only uses the paths and labels typed into the form to generate links.

1. Test URL Registration

Click the smoke-test link first. It should launch Odon or focus an already running Odon window. This only tests the operating-system URL handler; it does not open a project ROI.

Open odon://open Open Installed Synthetic Example

If nothing happens, the app is not registered as the handler for the odon:// scheme. Reinstall the packaged app or open the packaged macOS app once so the OS can register it. The installed example link requires a packaged build that includes the synthetic example files.

Suggested Fixture Test

To test with the repository fixture, open Odon, add fixtures/synthetic_5ch.ome.zarr from the Project panel, save the project, then use that saved project JSON path here. The ROI value is usually synthetic_5ch.ome.zarr, and useful channel names are DAPI, CD3, PanCK, Ki67, and Collagen.
